### ✅ ENDPOINT 1 — Register
```
POST /auth/register
Body: { fullName: string, email: string, password: string, role: 1|2|3 }
      role: 1=Buyer  2=Seller(Provider)  3=Admin
Response: { success: true, message: "User Registered Successfully" }
Error: 409 → email already exists
```

### ✅ ENDPOINT 2 — Login
```
POST /auth/login
Body: { email: string, password: string }
Response: { accessToken: string, refreshToken: string, role: "Buyer"|"Provider"|"Admin" }
NOTE: "Provider" = Seller in UI — map it!
Error: 401 → invalid credentials
```

### ✅ ENDPOINT 3 — Refresh Token
```
POST /auth/refresh
Body: { refreshToken: string }
Response: { accessToken: string, refreshToken: string }
Error: 401 → refresh failed → logout user
```

### ✅ ENDPOINT 4 — Get Categories
```
GET /categories
Auth: Not required
Response: {
  success: true,
  data: {
    items: [{ id, slug, nameAr, isActive, sortOrder, createdAt }]
  }
}
```

### ✅ ENDPOINT 5 — Home Sections
```
GET /services/home/{no}
Param: no = number of services per section (use 4)
Auth: Not required
Response: {
  topDiscounts: ServiceCard[],
  recommended: ServiceCard[],
  mostPurchased: ServiceCard[],
  regular: ServiceCard[]
}
ServiceCard: { id, title, providerName, basePrice, rating, imageUrl }
```

### ✅ ENDPOINT 6 — Get All Services
```
GET /services
Query: page(default 1), pageSize(default 10), categoryId?, isRecommended?, topDiscounts?, mostPurchased?
Auth: Not required
Response: { items: ServiceCard[], page, pageSize, totalPages }
```

### ✅ ENDPOINT 7 — Search Services
```
GET /services/name/{name}
Path: name = search keyword (URL encoded)
Query: pageNumber, pageSize
Auth: Not required
Response: { items: [{ id, title, basePrice, providerName, thumbnailImageUrl }] }
⚠️  Field is "thumbnailImageUrl" here not "imageUrl"
```

### ✅ ENDPOINT 8 — Get Service By ID
```
GET /services/{id}
Auth: Not required
Response: {
  id, title, serviceDescription, basePrice, address,
  provider: { id, fullName, rating, reviewsNo },
  optionGroups: [{
    id, name, isRequired, allowMultiple,
    options: [{ id, name, priceAdjustment }]
  }],
  images: [{ id, imageUrl }]
}
```

### ✅ ENDPOINT 9 — Request a Service
```
POST /orders/request
Auth: Required (Bearer token)
Body: {
  serviceId: number,
  customizationText: string,
  customRequestImage?: string,
  quantity: number,
  optionGroups: [{ groupId: number, items: [{ optionId: number, quantity: number }] }]
}
Response: { requestId: string, totalPrice: number, requestDate: string }
```

### ✅ ENDPOINT 10 — Get All Orders (Buyer)
```
GET /orders
Auth: Required
Query: status?, page?, pageSize?
Response: {
  items: [{
    id, serviceName, customizationDetails, totalPrice,
    status, providerName, createdAt,
    options: [{ groupOption, option, quantity }]
  }]
}
Status values: "Request" | "RePriced" | "AcceptedPrice" | "Accepted" |
               "PaymentFailed" | "Paid" | "InProgress" | "ReadyForShipping" |
               "OutForDelivery" | "Delivered" | "Completed" |
               "CancelledByBuyer" | "CancelledBySeller" | "Rejected" | "Expired"
```

GET /order/payment-summary       ← SINGULAR "order" not "orders"!

### Framer Motion — Stats Counter
```tsx
// Dashboard stat numbers count up from 0
const MotionNumber = ({ value }: { value: number }) => {
  const count = useMotionValue(0);
  const rounded = useTransform(count, Math.round);
  useEffect(() => {
    animate(count, value, { duration: 1.5, ease: 'easeOut' });
  }, [value]);
  return <motion.span>{rounded}</motion.span>;
};
```

## ⚡ CRITICAL GOTCHAS — DON'T MISS THESE

```
1. Login response role "Provider" → map to "Seller" in UI everywhere
2. /order/payment-summary is SINGULAR (not /orders/)
3. Payment proof = base64 JSON, NOT file upload
4. Search URL = /services/name/{name} (name in PATH not query)
5. Seller services response = PascalCase (Id, Title, Image...)
6. Payment summary response = PascalCase (Services, Address...)
7. serviceDescription (not description) in service details
8. priceAdjustment (not price) in option details
9. thumbnailImageUrl (not imageUrl) in search results
10. register role = number (1/2/3), login role = string
11. JWT sub is userId, email is email — no /users/me endpoint exists
12. basePrice in service details is often 0 — total = sum of priceAdjustments
13. All Not-Ready endpoints return empty → show Coming Soon (never crash)
```
